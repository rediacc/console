# Installazione

Installare `rdc` richiede tre passi: apri la pagina di installazione, scegli il tuo sistema operativo, incolla il comando nel terminale. Di solito termina in un minuto o due.

## Guarda il tutorial

![Tutorial: Installazione](/assets/tutorials/tutorial-installation.cast)

## I tre passi

![Panoramica dei tre passi](/img/tutorials/tutorial-installation/slide-1.svg)

1. Apri la [pagina di installazione](/it/install).
2. Scegli il tuo sistema operativo.
3. Copia il comando di installazione e incollalo nel tuo terminale.

## Passo 1: Installare il CLI

<TutorialStep number={1} />

La pagina di installazione genera il comando giusto per te. Su Windows (PowerShell), il one-liner equivalente è:

```powershell
iwr -useb https://www.rediacc.com/install.ps1 | iex
```

> Il prefisso `time` è un trucco della shell che mostra quanto tempo ha impiegato un comando. Lo usiamo in tutta questa serie così puoi vedere la velocità reale di ogni passo. È opzionale: rimuovilo se non lo vuoi.

## Passo 2: Verificare l'installazione

<TutorialStep number={2} />

Esegui `rdc doctor` per controllare Node, SSH e le restanti dipendenze di `rdc` e segnalare eventuali mancanze.

## Perché `rdc` si trova sul tuo laptop

![rdc sul tuo laptop, renet sul server](/img/tutorials/tutorial-installation/slide-2.svg)

`rdc` è la CLI sul tuo laptop. Il server esegue un componente separato chiamato `renet`, che `rdc` provisiona e gestisce via SSH. Non hai mai bisogno di connetterti via SSH a un server manualmente: `rdc` se ne occupa per te. Questo è il modello: uno strumento sulla tua macchina, niente da gestire direttamente sul server.

Affronteremo la configurazione di renet nei prossimi due tutorial. È più complessa dell'installazione del CLI, ma la fai una sola volta per macchina.
