# Créer votre premier dépôt

Ainsi, un dépôt Rediacc est un seul fichier chiffré sur votre serveur. Montez-le et vous obtenez un dossier avec son propre daemon Docker et ses propres données d'application, isolé de tout le reste sur la machine.

Pensez-y comme une clé USB pour la production : un fichier au repos, un serveur en exécution.

## Fichier sur disque, environnement une fois monté

![Un fichier chiffré se monte en dossier isolé](/img/tutorials/tutorial-create-repo/slide-1.svg)

Sur disque, c'est une seule image chiffrée. Une fois montée, vous obtenez :

- Un daemon Docker dédié (distinct de celui de l'hôte)
- Les données d'application à l'intérieur du volume chiffré
- Des adresses IP loopback qui n'entrent pas en conflit avec quoi que ce soit d'autre sur la machine

Les dépôts se déplacent. Copiez-en un entre machines, sauvegardez-le, ou forkez-le (les forks sont quasi-instantanés et en temps constant : un dépôt de 100 Go se forke aussi vite qu'un de 1 Go). Chaque dépôt est isolé de tous les autres sur le même serveur. Cet isolement est le but recherché.
