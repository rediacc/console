# SSH キーの設定

SSH キーを使うと、パスワードなしでサーバーにアクセスできます。これは `rdc` とは独立した通常の SSH です: ラップトップでキーを一度生成し、各サーバーで承認するだけです。

## チュートリアル動画

![チュートリアル: SSH キーの設定](/assets/tutorials/tutorial-ssh-keys.cast)

## 仕組み

![キーを生成して、各サーバーで承認する](/img/tutorials/tutorial-ssh-keys/slide-1.svg)

1. **生成**: ラップトップで SSH キーを生成します。一度だけ行います。
2. **承認**: `ssh-copy-id` を使って各サーバーで承認します。新しいサーバーを追加するたびに繰り返します。

`<user>` と `<server-ip>` は、お使いのサーバーの SSH ユーザーと IP アドレスに置き換えてください。この最後の一度だけサーバーのパスワードを求められます。これ以降、パスワード認証は不要になります。
