# ブラウザで VS Code を使う

ターミナルよりエディタが好みですか？1つのコマンドで、リポジトリのサンドボックス内からフル機能の VS Code をサービス提供できます。Linux、macOS、Windows、タブレットを含む任意のブラウザからアクセス可能です。ローカルに何もインストールする必要はありません。

## チュートリアル動画

![チュートリアル: ブラウザで VS Code を使う](/assets/tutorials/tutorial-vscode-browser.cast)

## 仕組み

![エディタはサンドボックス内で動作する](/img/tutorials/tutorial-vscode-browser/slide-1.svg)

エディタはラップトップ上では動きません。サーバー上の**リポジトリのサンドボックス内**で動作し、ブラウザは SSH トンネル経由でアクセスします。ファイルツリー、統合ターミナル、そこから起動するものはすべて、リポジトリと同じ視点から見えます。それ以外は見えません。

## ステップ1: 接続する

<TutorialStep number={1} />

初回接続時に、マシン上にオープンソースのエディタサーバーがインストールされます（チェックサム検証済み、読み取り専用で共有）。以降の再接続では、実行中のサーバーが再利用されます。

いつでも管理できます。

```bash
rdc vscode serve status -r my-app -m server-1
rdc vscode serve stop -r my-app -m server-1
```

## ステップ2: ブラウザで作業する

フォルダやファイルを作成、編集、保存すると、すべてリポジトリに直接反映されます。統合ターミナルも使えます。`rdc term` と同じサンドボックスのルールに従います。

## ステップ3: 壁は本物

<TutorialStep number={5} />

拒否はカーネルからきます。設定からではありません。サンドボックス内で動くもの（エディタ、ターミナル、ビルドツール、AI エージェント）はすべてリポジトリの壁で止まります。

> **ローカルの VS Code を使う場合は？** `rdc vscode connect -r my-app -m server-1`（`--browser` なし）で、インストール済みの VS Code が Remote-SSH 経由で同じリポジトリを開きます。
