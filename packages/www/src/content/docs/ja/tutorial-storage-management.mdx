# ストレージ管理

ディスク不足に悩まされることはもうありません。コンテナが動き続けたまま満杯のリポジトリを拡張し、削除したブロックをプールに返却し、その両方を自動的にマシンに任せます。

## チュートリアル動画

![チュートリアル: ストレージ管理](/assets/tutorials/tutorial-storage-management.cast)

## 1つのプール、成長するリポジトリ

![1つのプール、成長するリポジトリ](/img/tutorials/tutorial-storage-management/slide-1.svg)

各リポジトリは共有データストアプール内の1つのファイルです。リポジトリはオンラインで成長できます。リポジトリ内のデータを削除してもそれだけではブロックは解放されません。**トリム**がブロックをプールに返す役割を担います。
