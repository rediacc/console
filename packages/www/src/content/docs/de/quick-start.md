# Schnellstart

Willkommen bei Rediacc. Dieser Abschnitt führt Sie durch alles, was Sie benötigen, um verschlüsselte, isolierte Infrastruktur auf Ihren eigenen Servern bereitzustellen.

## Erste Schritte

Folgen Sie diesen Seiten der Reihe nach:

1. **[Voraussetzungen](/de/docs/requirements)** -- Unterstützte Betriebssysteme und Voraussetzungen für Ihre Workstation und entfernte Server.

2. **[Installation](/de/docs/installation)** -- Installieren Sie die `rdc`-CLI unter Linux, macOS oder Windows.

3. **[Schritt-für-Schritt-Anleitung](/de/docs/guide)** -- Erstellen Sie einen Kontext, fügen Sie Maschinen hinzu, erstellen Sie verschlüsselte Repositories und stellen Sie Dienste bereit.

4. **[Nach der Installation](/de/docs/post-installation)** -- Konfigurieren Sie Autostart, verstehen Sie die Kontext-Konfiguration und beheben Sie häufige Probleme.

5. **[Werkzeuge](/de/docs/tools)** -- Dateisynchronisation, SSH-Terminal, VS Code-Integration, CLI-Updates und Diagnose.

## Was ist Rediacc?

Rediacc stellt containerisierte Dienste auf entfernten Servern bereit, die Sie kontrollieren. Alles wird im Ruhezustand mit LUKS verschlüsselt, jedes Repository erhält seinen eigenen isolierten Docker-Daemon, und die gesamte Orchestrierung erfolgt über SSH von Ihrer Workstation aus.

Keine Cloud-Konten. Keine SaaS-Abhängigkeiten. Ihre Daten bleiben auf Ihren Servern.
