# Monitoring

Ihre App ist deployed und gesichert. Stellen Sie jetzt sicher, dass alles gesund bleibt. Das macht `rdc`. `rdc` gibt Ihnen ein vollständiges Bild jedes Servers (Zustand, Container, Repos) von Ihrem Laptop aus.

## Tutorial ansehen

![Tutorial: Monitoring](/assets/tutorials/tutorial-monitoring.cast)

## Drei Dinge, die Sie prüfen können

![Zustand, Container, Repos](/img/tutorials/tutorial-monitoring/slide-1.svg)

## Zustand: Systeminformationen

<TutorialStep number={1} />

Sie sehen Systemuptime, Festplattennutzung und Speicherstatus. Sollte etwas nicht stimmen, sehen Sie es sofort.

## Container

<TutorialStep number={2} />

Sie erhalten Name, Status, Zustand, CPU und Speicher für jeden Container, plus welches Repo ihn besitzt.

## Repos

<TutorialStep number={3} />

Das zeigt jedes Repo mit seiner Größe, Einhängestatus, Docker-Status und Festplattennutzung.

## Alles auf einmal

<TutorialStep number={4} />

`query` ohne Flags gibt Ihnen das vollständige Bild: Systeminformationen, Repos und Container. Fügen Sie `--system`, `--containers`, `--repositories`, `--services`, `--network` oder `--block-devices` hinzu, um es auf genau diesen Bereich einzugrenzen.
