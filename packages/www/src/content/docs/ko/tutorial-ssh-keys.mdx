# SSH 키 구성

SSH 키를 사용하면 비밀번호 없이 서버에 접근할 수 있습니다. 이것은 `rdc`와 독립적인 일반 SSH입니다: 노트북에서 키를 한 번 생성한 다음 각 서버에서 승인하면 됩니다.

## 튜토리얼 보기

![Tutorial: SSH key configuration](/assets/tutorials/tutorial-ssh-keys.cast)

## 작동 방식

![키를 생성한 다음 각 서버에서 승인](/img/tutorials/tutorial-ssh-keys/slide-1.svg)

1. 노트북에서 SSH 키를 **생성**합니다. 딱 한 번.
2. `ssh-copy-id`로 각 서버에서 **승인**합니다. 새 서버마다 반복.

## 1단계: 키 생성

사용하고 싶은 키가 이미 있다면 건너뜁니다. 없다면:

<TutorialStep number={1} />

`ed25519`는 현대적인 기본값입니다. 작고, 빠르고, 잘 지원됩니다.

## 2단계: 첫 번째 서버 승인

<TutorialStep number={2} />

`<user>`와 `<server-ip>`를 서버의 SSH 사용자와 IP로 교체하세요. 서버 비밀번호를 마지막으로 한 번 입력하라는 요청이 있습니다. 이후 비밀번호 인증이 더 이상 필요하지 않습니다.

## 3단계: 나머지 서버 승인

<TutorialStep number={3} />

관리하는 모든 머신에 대해 `ssh-copy-id`를 반복하세요. SSH에 필요한 것은 그것뿐입니다. 다음 튜토리얼에서 `rdc`가 이 키를 자동으로 사용합니다.
