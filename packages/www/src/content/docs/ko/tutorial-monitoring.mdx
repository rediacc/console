# 모니터링

앱이 배포되고 백업되었습니다. 이제 건강한 상태를 유지해야 합니다. 그것이 `rdc`의 역할입니다. `rdc`는 노트북에서 모든 서버의 전체 모습(상태, 컨테이너, 저장소)을 파악할 수 있게 해줍니다.

## 튜토리얼 보기

![Tutorial: Monitoring](/assets/tutorials/tutorial-monitoring.cast)

## 확인할 수 있는 세 가지

![Health, containers, repos](/img/tutorials/tutorial-monitoring/slide-1.svg)

## 상태 확인: 시스템 정보

<TutorialStep number={1} />

시스템 가동 시간, 디스크 사용량, 스토리지 상태를 표시합니다. 문제가 있으면 바로 알 수 있습니다.

## 컨테이너

<TutorialStep number={2} />

각 컨테이너의 이름, 상태, 헬스, CPU, 메모리와 어떤 저장소가 소유하는지 확인할 수 있습니다.

## 저장소

<TutorialStep number={3} />

모든 저장소의 크기, 마운트 상태, Docker 상태, 디스크 사용량을 확인할 수 있습니다.

## 한 번에 모든 것 확인

<TutorialStep number={4} />

필터 없이 `query` 명령을 사용하면 전체 모습을 볼 수 있습니다: 시스템 정보, 저장소, 컨테이너. `--system`, `--containers`, `--repositories`, `--services`, `--network`, `--block-devices` 플래그를 추가하여 특정 섹션만 필터링할 수 있습니다.

## 기본 과정 완료

설치, 구성, 배포, 포크, 라이브 전환, 자동 시작, 백업, 모니터링을 모두 터미널에서, 여러분 자신의 서버에서 할 수 있게 되었습니다.
