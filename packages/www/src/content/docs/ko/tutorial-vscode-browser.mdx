# 브라우저에서 VS Code 사용하기

터미널보다 에디터가 편하신가요? 명령 하나로 저장소 샌드박스 안에서 VS Code 전체를 서빙하고, Linux, macOS, Windows, 심지어 태블릿 어느 브라우저에서든 접근할 수 있습니다. 로컬에 아무것도 설치할 필요 없습니다.

## 튜토리얼 보기

![Tutorial: VS Code in your browser](/assets/tutorials/tutorial-vscode-browser.cast)

## 작동 방식

![The editor lives in the sandbox](/img/tutorials/tutorial-vscode-browser/slide-1.svg)

에디터는 노트북이 아닌 서버의 **저장소 샌드박스 내부**에서 실행됩니다. 브라우저는 SSH 터널을 통해 접근합니다. 파일 트리, 통합 터미널, 그 안에서 실행되는 모든 것은 저장소가 보는 것과 동일한 뷰를 갖습니다. 그 이상은 없습니다.

## 1단계: 연결

<TutorialStep number={1} />

첫 연결 시 머신에 오픈소스 에디터 서버가 설치됩니다(체크섬 검증, 읽기 전용으로 공유). 이후 재연결은 실행 중인 서버를 재사용합니다.

언제든지 관리할 수 있습니다:

```bash
rdc vscode serve status -r my-app -m server-1
rdc vscode serve stop -r my-app -m server-1
```

## 2단계: 브라우저에서 작업하기

폴더와 파일을 만들고, 편집하고, 저장하면 모든 내용이 바로 저장소에 반영됩니다. 통합 터미널도 동작하며, `rdc term`과 동일한 샌드박스를 따릅니다.

차단은 설정이 아닌 커널에서 옵니다. 샌드박스 내부에서 실행되는 것은 에디터든 터미널이든 빌드 도구든 AI 에이전트든, 무엇이든 저장소의 경계에서 멈춥니다.

> **로컬 VS Code를 원하신다면?** `rdc vscode connect -r my-app -m server-1` (`--browser` 없이)을 실행하면 Remote-SSH를 통해 설치된 VS Code에서 동일한 저장소가 열립니다.
