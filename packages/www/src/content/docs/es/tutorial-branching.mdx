# Branching: Commit y rollback

Versiona tus repositorios igual que el código. Un **commit** congela un fork en una imagen inmutable; un **branch** le pone nombre; **checkout** convierte cualquier commit en un fork nuevo con escritura. En este tutorial hacemos commit de un estado conocido como bueno, borramos una tabla real de PostgreSQL y recuperamos todas las filas en segundos.

## Ver el tutorial

![Tutorial: Branching, commit y rollback](/assets/tutorials/tutorial-branching.cast)

## El modelo

![Commits, branches, checkout](/img/tutorials/tutorial-branching/slide-1.svg)

Los commits nunca cambian: ni siquiera se pueden montar. Los forks nunca esperan: el checkout es un clon reflink casi instantáneo. El modelo completo está en la [guía de branching](/es/docs/repo-branching).
