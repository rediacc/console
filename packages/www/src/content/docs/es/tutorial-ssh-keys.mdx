# Configuración de claves SSH

Las claves SSH te permiten acceder a tus servidores sin contraseñas. Esto es SSH estándar, independiente de `rdc`: generas una clave en tu laptop, una sola vez, y luego la autorizas en cada servidor.

## Ver el tutorial

![Tutorial: Configuración de claves SSH](/assets/tutorials/tutorial-ssh-keys.cast)

## Cómo funciona

![Generar una clave, luego autorizarla en cada servidor](/img/tutorials/tutorial-ssh-keys/slide-1.svg)

1. **Genera** una clave SSH en tu laptop. Una vez, para siempre.
2. **Autorízala** en cada servidor con `ssh-copy-id`. Se repite por cada nuevo servidor.

## Paso 1: Generar una clave

Si ya tienes una clave que quieras usar, pasa al siguiente paso. De lo contrario:

<TutorialStep number={1} />

`ed25519` es el estándar moderno: pequeño, rápido y ampliamente compatible.

## Paso 2: Autorizar tu primer servidor

<TutorialStep number={2} />

Reemplaza `<user>` y `<server-ip>` con el usuario SSH y la IP de tu servidor. Se te pedirá la contraseña del servidor por última vez. Después de esto, la autenticación por contraseña ya no será necesaria.

## Paso 3: Autorizar cualquier otro servidor

<TutorialStep number={3} />

Repite `ssh-copy-id` para cada máquina que administres. Eso es todo lo que SSH necesita. En el siguiente tutorial, `rdc` usará esta clave automáticamente.
