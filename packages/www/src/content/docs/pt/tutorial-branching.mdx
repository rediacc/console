# Branching: Commit e Rollback

Versione os seus repositórios como código. Um **commit** congela um fork numa imagem imutável; um **branch** dá-lhe nome; o **checkout** transforma qualquer commit num novo fork gravável. Neste tutorial fazemos commit de um estado estável, apagamos uma tabela PostgreSQL real e recuperamos todas as linhas em segundos.

## Ver o tutorial

![Tutorial: Branching, commit and rollback](/assets/tutorials/tutorial-branching.cast)

## O modelo

![Commits, branches, checkout](/img/tutorials/tutorial-branching/slide-1.svg)

Os commits nunca mudam: recusam-se mesmo a montar. Os forks nunca esperam: o checkout é um clone reflink quase instantâneo. O modelo mental completo está no [guia de branching](/pt/docs/repo-branching).
