# Configuração de Chave SSH

As chaves SSH permitem-lhe aceder aos seus servidores sem palavras-passe. Trata-se de SSH standard, independente do `rdc`: gera uma chave no seu computador portátil, e depois autoriza-a em cada servidor.

## Ver o tutorial

![Tutorial: SSH key configuration](/assets/tutorials/tutorial-ssh-keys.cast)

## Como funciona

![Gerar uma chave, depois autorizá-la em cada servidor](/img/tutorials/tutorial-ssh-keys/slide-1.svg)

1. **Gere** uma chave SSH no seu computador portátil. Uma vez, para sempre.
2. **Autorize-a** em cada servidor com `ssh-copy-id`. Repita para cada novo servidor.

## Passo 1: Gerar uma chave

Se já tem uma chave que quer usar, avance. Caso contrário:

<TutorialStep number={1} />

`ed25519` é o padrão moderno -- pequeno, rápido e bem suportado.

## Passo 2: Autorizar o seu primeiro servidor

<TutorialStep number={2} />

Substitua `<user>` e `<server-ip>` pelo utilizador SSH e IP do seu servidor. Ser-lhe-á pedida a palavra-passe do servidor uma última vez. A partir daí, a autenticação por palavra-passe deixa de ser necessária.

## Passo 3: Autorizar os outros servidores

<TutorialStep number={3} />

Repita `ssh-copy-id` para cada máquina que gere. É tudo o que o SSH precisa. No próximo tutorial, o `rdc` usa esta chave automaticamente.
