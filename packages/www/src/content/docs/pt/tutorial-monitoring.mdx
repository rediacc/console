# Monitorização

A sua aplicação está implementada e com backup. Agora mantenha-a saudável. É isto que o `rdc` faz. O `rdc` dá-lhe a imagem completa de qualquer servidor (saúde, contentores, repositórios) a partir do seu computador portátil.

## Ver o tutorial

![Tutorial: Monitoring](/assets/tutorials/tutorial-monitoring.cast)

## Três coisas que pode verificar

![Health, containers, repos](/img/tutorials/tutorial-monitoring/slide-1.svg)

## Saúde: informação do sistema

<TutorialStep number={1} />

Mostra o tempo de atividade do sistema, uso do disco e estado do armazenamento. Se algo estiver errado, diz-lho.

## Contentores

<TutorialStep number={2} />

Obtém nome, estado, saúde, CPU e memória para cada contentor, mais qual o repositório a que pertence.

## Repositórios

<TutorialStep number={3} />

Mostra cada repositório com o seu tamanho, estado de montagem, estado Docker e uso do disco.

## Tudo de uma vez

<TutorialStep number={4} />

O comando `query` sem filtros dá-lhe a imagem completa: informação do sistema, repositórios e contentores. Adicione `--system`, `--containers`, `--repositories`, `--services`, `--network` ou `--block-devices` para restringir-se a essa secção.
