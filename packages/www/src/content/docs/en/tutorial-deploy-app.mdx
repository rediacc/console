# Deploying Your First App

You have an empty repository. `rdc` ships built-in templates so you can spin up real apps without writing a `docker-compose` from scratch. Three steps: pick a template, apply it, run it.

## Watch the tutorial

![Tutorial: Deploying your first app](/assets/tutorials/tutorial-deploy-app.cast)

## Pick, apply, run

![Pick a template, apply it, run it](/img/tutorials/tutorial-deploy-app/slide-1.svg)

## Step 1: Pick

<TutorialStep number={1} />

You'll see ready-made setups for common apps: Postgres, Redis, web servers, and more.

## Step 2: Apply

<TutorialStep number={2} />

This copies the template's files into the repo.

## Step 3: See what landed

<TutorialStep number={3} />

Two files: `docker-compose.yml` describing the containers, and `Rediaccfile` defining what happens when the app starts and stops (its `up` and `down` lifecycle hooks). The `2>/dev/null` hides the connection logs so only the file listing shows.

## Step 4: Run

<TutorialStep number={4} />

The CLI starts the containers, mounts the encrypted volume, and wires up an isolated Docker daemon for this repo. (If you're already inside the repo's sandbox via VS Code, `renet dev up` is an equivalent in-sandbox alternative.)

## Step 5: Verify

<TutorialStep number={5} />

`docker ps` here lists only this repo's containers. Other repos on the same server have their own Docker daemons and are completely invisible from this one.
