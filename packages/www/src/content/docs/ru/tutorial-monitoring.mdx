# Мониторинг

Приложение развёрнуто и резервно скопировано. Теперь поддерживайте его в здоровом состоянии. Это то, что делает `rdc`. `rdc` даёт полную картину любого сервера (состояние, контейнеры, репозитории) с вашего ноутбука.

## Смотреть урок

![Tutorial: Monitoring](/assets/tutorials/tutorial-monitoring.cast)

## Три аспекта для проверки

![Health, containers, repos](/img/tutorials/tutorial-monitoring/slide-1.svg)

## Состояние: системная информация

<TutorialStep number={1} />

Вы увидите время работы системы, использование диска и статус хранилища. Если что-то не так, вы это заметите.

## Контейнеры

<TutorialStep number={2} />

Вы получаете имя, статус, здоровье, CPU и память для каждого контейнера, а также какому репозиторию он принадлежит.

## Репозитории

<TutorialStep number={3} />

Вы видите все репозитории: их размер, статус монтирования, статус Docker и использование диска.

## Всё одним запросом

<TutorialStep number={4} />

Команда `query` без флагов показывает полную картину: системная информация, репозитории и контейнеры. Добавьте `--system`, `--containers`, `--repositories`, `--services`, `--network` или `--block-devices`, чтобы получить только нужный раздел.
