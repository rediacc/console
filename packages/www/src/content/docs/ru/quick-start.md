# Быстрый старт

Разверните зашифрованную изолированную контейнерную среду на собственном сервере за 5 минут. Облачные аккаунты и SaaS-зависимости не требуются.

## Предварительные требования

- Рабочая станция на Linux или macOS
- Удалённый сервер (Ubuntu 24.04+, Debian 12+ или Fedora 43+) с доступом по SSH и привилегиями sudo
- Пара SSH-ключей (например, `~/.ssh/id_ed25519`)

## 1. Установка CLI

```bash
curl -fsSL https://get.rediacc.com | sh
```

## 2. Создание конфигурации

```bash
rdc config init my-infra --ssh-key ~/.ssh/id_ed25519
```

## 3. Добавление сервера

```bash
rdc config add-machine server-1 --ip <your-server-ip> --user <your-ssh-user>
```

## 4. Подготовка сервера

```bash
rdc config setup-machine server-1
```

Эта команда устанавливает Docker, cryptsetup и бинарный файл renet на ваш сервер.

## 5. Создание зашифрованного репозитория

```bash
rdc repo create my-app -m server-1 --size 5G
```

## 6. Развёртывание сервисов

Подключите репозиторий, создайте в нём файлы `docker-compose.yml` и `Rediaccfile`, затем запустите:

```bash
rdc repo up my-app -m server-1 --mount
```

## 7. Проверка

```bash
rdc machine containers server-1
```

Вы должны увидеть свои работающие контейнеры.

## Что такое Rediacc?

Rediacc развёртывает контейнеризированные сервисы на удалённых серверах, которые вы контролируете. Все данные шифруются в состоянии покоя с помощью LUKS, каждый репозиторий получает собственный изолированный Docker-демон, а вся оркестрация происходит по SSH с вашей рабочей станции.

Никаких облачных аккаунтов. Никаких SaaS-зависимостей. Ваши данные остаются на ваших серверах.
