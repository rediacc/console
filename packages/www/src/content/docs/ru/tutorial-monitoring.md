# Мониторинг

Приложение развёрнуто, работает и резервно скопировано. Теперь убедитесь, что всё остаётся здоровым. `rdc` даёт полную картину любого сервера (состояние, контейнеры, репозитории) с вашего ноутбука.

## Состояние: системная информация

Начните с системного вида:

```bash
time rdc machine query --name my-server --system
```

Отображаются время работы системы, использование диска и статус хранилища. Если что-то не так, команда сообщит об этом.

## Контейнеры

Чтобы увидеть все запущенные контейнеры на всех репозиториях машины:

```bash
time rdc machine query --name my-server --containers
```

Вы получите имя, статус, здоровье, CPU и память для каждого контейнера, а также какому репозиторию он принадлежит.

## Репозитории

Для проверки репозиториев:

```bash
time rdc machine query --name my-server --repositories
```

Отображается каждый репозиторий с его размером, статусом монтирования, статусом Docker и использованием диска.

## Всё одним запросом

```bash
time rdc machine query --name my-server
```

Системная информация, репозитории, контейнеры — всё в одной команде. Та же команда `query` без фильтров возвращает полную картину; с `--system`, `--containers`, `--repositories`, `--services`, `--network` или `--block-devices` сужается до нужного раздела.

## Локальная проверка

`rdc doctor` проверяет локальную настройку (Node, SSH-ключ, `renet`, Docker), независимо от конкретного сервера:

```bash
time rdc doctor
```
