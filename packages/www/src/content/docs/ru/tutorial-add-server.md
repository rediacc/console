# Добавление первого сервера

Прежде чем добавить сервер, полезно понять, как работает `rdc`. Rediacc построен на двух инструментах: `rdc` на вашем ноутбуке и `renet` на сервере.

## Смотреть урок

![Tutorial: Adding your first server](/assets/tutorials/tutorial-add-server.cast)

## Зачем два инструмента?

![rdc on laptop, renet on server, SSH between](/img/tutorials/tutorial-add-server/slide-1.svg)

- **`rdc`**: CLI на вашем ноутбуке. Здесь вы вводите команды.
- **`renet`**: оркестратор на сервере. Он управляет шифрованием, Docker и изоляцией.

Когда вы выполняете команду локально, `rdc` подключается по SSH и запускает `renet` на сервере. Вам никогда не придётся вручную заходить на серверы по SSH: `rdc` сделает это за вас.

## Шаг 1: Регистрация сервера

Расскажите `rdc` о сервере. Замените имя, IP и пользователя на свои.

```bash
time rdc config machine add --name my-server --ip 192.168.1.100 --user deploy
```

## Шаг 2: Подготовка сервера

Эта команда устанавливает `renet` и создаёт зашифрованное хранилище данных на сервере.

```bash
time rdc config machine setup --name my-server
```

После завершения сервер готов к размещению репозиториев.

## Где хранится конфигурация

Проверьте, что `rdc` знает о вашей настройке:

```bash
time rdc config show
```

Или откройте JSON-файл напрямую:

```bash
vim ~/.config/rediacc/rediacc.json
```

В этом одном файле хранится всё: машины, репозитории, SSH-ключ, учётные данные шифрования. Скопируйте его на другой ноутбук, и вы сможете управлять теми же серверами с него.
