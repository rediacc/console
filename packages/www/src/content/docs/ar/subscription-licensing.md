# الاشتراك والترخيص

تحتاج الأجهزة العاملة في عمليات النشر المحلية إلى ترخيص اشتراك لفرض حدود الموارد حسب الخطة. يقوم CLI تلقائياً بتسليم حزم التراخيص الموقّعة إلى الأجهزة البعيدة عبر SSH — دون الحاجة إلى تفعيل يدوي أو اتصال سحابي من جانب الخادم.

## نظرة عامة

1. سجّل الدخول باستخدام `rdc subscription login` (يفتح المتصفح للمصادقة)
2. استخدم أي أمر خاص بالجهاز — تتم معالجة التراخيص تلقائياً

عند تشغيل أمر يستهدف جهازاً (`rdc machine info`، `rdc repo up`، إلخ)، يتحقق CLI تلقائياً مما إذا كان الجهاز يملك ترخيصاً صالحاً. إذا لم يكن كذلك، يجلب واحداً من خادم الحساب ويسلّمه عبر SSH.

## تسجيل الدخول

```bash
rdc subscription login
```

يفتح المتصفح للمصادقة عبر تدفق رمز الجهاز. بعد الموافقة، يخزّن CLI رمز API محلياً في `~/.config/rediacc/api-token.json`.

| الخيار | مطلوب | الافتراضي | الوصف |
|--------|----------|---------|-------------|
| `-t, --token <token>` | لا | - | رمز API (يتخطى تدفق المتصفح) |
| `--server <url>` | لا | `https://account.rediacc.com` | عنوان خادم الحساب |

## التحقق من الحالة

```bash
# حالة مستوى الحساب (الخطة، الأجهزة)
rdc subscription status

# تضمين تفاصيل الترخيص من جهاز محدد
rdc subscription status -m hostinger
```

يعرض تفاصيل الاشتراك من خادم الحساب. مع `-m`، يتصل أيضاً بالجهاز عبر SSH ويعرض معلومات الترخيص الحالية.

## إعادة تحديث الترخيص إجبارياً

```bash
rdc subscription refresh -m <machine>
```

يُعيد إصدار ترخيص جديد ويسلّمه إلى الجهاز المحدد إجبارياً. هذا غير مطلوب عادةً — يتم تحديث التراخيص تلقائياً كل 50 دقيقة أثناء الاستخدام العادي لـ CLI.

## كيف يعمل

1. **تسجيل الدخول** يخزّن رمز API على محطة العمل الخاصة بك
2. **أي أمر خاص بالجهاز** يُطلق فحص ترخيص تلقائي عبر SSH
3. إذا كان الترخيص البعيد مفقوداً أو أقدم من 50 دقيقة، يقوم CLI بـ:
   - قراءة معرّف العتاد للجهاز البعيد عبر SSH
   - استدعاء API الحساب لإصدار ترخيص جديد
   - تسليم كل من ترخيص الجهاز وحزمة الاشتراك إلى الجهاز البعيد عبر SSH
4. ذاكرة تخزين مؤقت مدتها 50 دقيقة في الذاكرة تمنع جولات SSH المتكررة خلال نفس الجلسة

كل تفعيل جهاز يستهلك مقعداً واحداً في اشتراكك. لتحرير مقعد، قم بإلغاء تفعيل جهاز من بوابة الحساب.

## فترة السماح والتدهور

إذا انتهت صلاحية الترخيص ولم يمكن تحديثه خلال فترة السماح البالغة 3 أيام، تتدهور حدود موارد الجهاز إلى الإعدادات الافتراضية لخطة Community. بمجرد تحديث الترخيص (باستعادة الاتصال وتشغيل أي أمر `rdc`)، تُستعاد حدود الخطة الأصلية فوراً.
