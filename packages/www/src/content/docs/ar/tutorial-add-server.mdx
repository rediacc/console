# إضافة خادمك الأول

أداتان: `rdc` على حاسوبك، و`renet` على الخادم. هذه هي بنية Rediacc بالكامل. إذن إليك كيف تتصل قبل إضافة خادم.

## شاهد الدرس التعليمي

![Tutorial: Adding your first server](/assets/tutorials/tutorial-add-server.cast)

## لماذا أداتان؟

![rdc on laptop, renet on server, SSH between](/img/tutorials/tutorial-add-server/slide-1.svg)

- **`rdc`** هي أداة CLI على حاسوبك. هنا تكتب أوامرك.
- **`renet`** هو المنسق على الخادم. يدير التشفير وDocker والعزل.

عندما تشغّل أمرًا محليًا، تتصل `rdc` عبر SSH وتُنفّذ `renet` على الخادم. لن تحتاج أبدًا للاتصال بخوادمك يدويًا عبر SSH؛ `rdc` تفعل ذلك نيابةً عنك.

## الخطوة 1: تسجيل الخادم

<TutorialStep number={1} />

لا حاجة لـ `rdc config init`. يُنشأ ملف الإعداد تلقائيًا عند الاستخدام الأول، وتستخدم `rdc` مفتاح `~/.ssh` العادي كخيار احتياطي. اختيار المفتاح متعمد ومبسط بقدر كبير - إذا احتجت إلى إعادة توجيه الوكيل أو مضيف قفز، فقم بإعدادها في `~/.ssh/config` كالمعتاد؛ `rdc` ستقتطفها تلقائيًا.
