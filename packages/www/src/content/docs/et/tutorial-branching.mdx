# Hargnemine -- commit ja tagasipööramine

Versiooni oma repositooriumid nagu koodi. **Commit** külmutab forki muutumatuks kujutiseks; **branch** annab sellele nime; **checkout** muudab iga commit'i uuesti värskeks kirjutatavaks forkiks. Selles juhendis teeme teadaolevalt töötavast olekust commit'i, langetame päris PostgreSQL tabeli ja saame kõik read tagasi sekunditega.

## Vaata juhendvideot

![Tutorial: Branching, commit and rollback](/assets/tutorials/tutorial-branching.cast)

## Mudel

![Commit'id, branch'id, checkout](/img/tutorials/tutorial-branching/slide-1.svg)

Commit'id ei muutu kunagi -- need keelduvad isegi ühendamisest. Forkid ei oota kunagi -- checkout on peaaegu kohene reflink-kloon. Täielik mõttemudel on kirjas [hargnemise juhendis](/et/docs/repo-branching).
