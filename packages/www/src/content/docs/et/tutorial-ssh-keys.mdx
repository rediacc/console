# SSH võtme seadistamine

SSH võtmed võimaldavad sul serveritele paroolideta ligi pääseda. See on tavaline SSH, sõltumatu `rdc`-st: genereerid ühe võtme oma sülearvutis ja autoriseerid selle seejärel igal serveril.

## Vaata juhendvideot

![Tutorial: SSH key configuration](/assets/tutorials/tutorial-ssh-keys.cast)

## Kuidas see toimib

![Genereeri võti, seejärel autoriseeri see igal serveril](/img/tutorials/tutorial-ssh-keys/slide-1.svg)

1. **Genereeri** SSH võti oma sülearvutis. Üks kord, igaveseks.
2. **Autoriseeri** see igal serveril kasutades `ssh-copy-id`. Korda iga uue serveri puhul.

## 1. samm: genereeri võti

Kui sul on juba võti, mida soovid kasutada, jäta see samm vahele. Vastasel juhul:

<TutorialStep number={1} />

`ed25519` on kaasaegne vaikimisi valik: väike, kiire ja laialt toetatud.

## 2. samm: autoriseeri esimene server

<TutorialStep number={2} />

Asenda `<user>` ja `<server-ip>` oma serveri SSH kasutajanime ja IP-aadressiga. Sinult küsitakse serveri parooli viimast korda. Pärast seda ei ole parooliga autentimine enam vajalik.

## 3. samm: autoriseeri ülejäänud serverid

<TutorialStep number={3} />

Korda `ssh-copy-id` kõikide masinate puhul, mida haldad. See ongi kõik, mida SSH vajab. Järgmises juhendis kasutab `rdc` seda võtit automaatselt.
