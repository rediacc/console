# Esimese serveri lisamine

Kaks tööriista: `rdc` sinu sülearvutis ja `renet` serveris. See on kogu Rediacc arhitektuur. Siin on kirjeldus, kuidas need enne serveri lisamist ühenduvad.

## Vaata juhendvideot

![Tutorial: Adding your first server](/assets/tutorials/tutorial-add-server.cast)

## Miks kaks tööriista?

![rdc on laptop, renet on server, SSH between](/img/tutorials/tutorial-add-server/slide-1.svg)

- **`rdc`** on CLI sinu sülearvutis. Siin sisestad käsud.
- **`renet`** on orkestreerimistööriist serveris. See haldab krüpteerimist, Dockerit ja isolatsiooni.

Kui käivitad käsu kohalikult, ühendub `rdc` üle SSH ja täidab `renet`-i serveris. Sa ei pea kunagi käsitsi serverisse SSH kaudu ühenduma. `rdc` teeb seda sinu eest.

## 1. samm: registreeri server

<TutorialStep number={1} />

`rdc config init` ei ole vajalik. Konfiguratsioonifail luuakse automaatselt esimesel kasutamisel ja `rdc` kasutab tagavaravalikuna sinu standardset `~/.ssh` võtit. Võtmete valik on teadlikult minimaalne -- kui sa vajad agent forwarding-ut või jump host-i, seadista need `~/.ssh/config` failist nagu tavaliselt; `rdc` võtab need kasutusele.

## 2. samm: valmista see ette

<TutorialStep number={2} />

Sinu server on repositooriumite majutamiseks valmis.

## Kus konfiguratsioon asub

CLI salvestab kõik üheainukesse `~/.config/rediacc/rediacc.json` failina lihtsa JSON-vormingus:

<TutorialStep number={3} />

See üksik fail sisaldab sinu masinaid, SSH võtit ja krüpteerimise mandaate. Kopeeri see teisele sülearvutile ja oled sealt kohe töövalmis.

## Mitu konfiguratsiooni

Saad pidada mitut konfiguratsiooni, ühe iga keskkonna kohta (tootmine, lavastus, arendus), ja lülituda nende vahel käskudel `--config` lipuga. Vaata [mitme konfiguratsiooni seadistamist](/et/docs/setup#multiple-configs).
