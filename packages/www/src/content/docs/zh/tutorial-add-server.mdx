# 添加第一台服务器

两个工具：笔记本上的 `rdc`，服务器上的 `renet`。这就是整个 Rediacc 架构。在添加服务器之前，先了解它们如何连接。

## 观看教程

![Tutorial: Adding your first server](/assets/tutorials/tutorial-add-server.cast)

## 为什么需要两个工具？

![rdc 在笔记本上，renet 在服务器上，SSH 连接两者](/img/tutorials/tutorial-add-server/slide-1.svg)

- **`rdc`** 是运行在笔记本上的 CLI，你在这里输入命令。
- **`renet`** 是服务器上的编排器，负责管理加密、Docker 和隔离环境。

当你在本地运行命令时，`rdc` 通过 SSH 连接并在服务器上执行 `renet`。你永远不需要手动 SSH 进入服务器，`rdc` 会替你完成这一切。

## 第一步：注册服务器

<TutorialStep number={1} />

无需执行 `rdc config init`。配置文件在首次使用时自动创建，`rdc` 会回退使用你标准的 `~/.ssh` 密钥。密钥选择有意保持最小化 — 如果需要代理转发或跳转主机，按常规在 `~/.ssh/config` 中进行配置；`rdc` 会自动识别。

服务器已准备好托管仓库。

## 配置文件存放在哪里

CLI 将所有内容存储在 `~/.config/rediacc/rediacc.json` 这一个纯 JSON 文件中：

<TutorialStep number={3} />

这个单一文件保存了你的机器列表、SSH 密钥和加密凭据。将它复制到另一台笔记本，即可从那台机器操作相同的服务器。

## 多个配置

你可以保存多个配置，每个环境（生产、预发布、开发）一个，并在任意命令中使用 `--config` 标志切换。参见[设置多个配置](/zh/docs/setup#multiple-configs)。
