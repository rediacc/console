# 添加您的第一个服务器

两个工具：您笔记本上的 `rdc`，服务器上的 `renet`。这就是整个 Rediacc 架构。现在让我们看看在添加服务器之前它们如何连接。

## 观看教程

![教程：添加您的第一个服务器](/assets/tutorials/tutorial-add-server.cast)

## 为什么需要两个工具？

![笔记本上的 rdc，服务器上的 renet，通过 SSH 连接](/img/tutorials/tutorial-add-server/slide-1.svg)

- **`rdc`** 是您笔记本上的命令行界面。您在这里输入命令。
- **`renet`** 是服务器上的编排工具。它管理加密、Docker 和隔离。

当您在本地运行命令时，`rdc` 通过 SSH 连接并在服务器上执行 `renet`。您永远不需要手动 SSH 到您的服务器。`rdc` 为您完成这一切。

## 步骤 1: 注册服务器

<TutorialStep number={1} />

不需要运行 `rdc config init`。配置文件在首次使用时会自动创建，`rdc` 会使用您的标准 `~/.ssh` 密钥。密钥选择有意保持最小化。如果您需要代理转发或跳转主机，请在 `~/.ssh/config` 中进行配置；`rdc` 会自动识别。

## 步骤 2: 完成配置

<TutorialStep number={2} />

服务器现在可以托管存储库了。

这个文件包含您的机器、SSH 密钥和加密凭证。将其复制到另一台笔记本电脑，您就可以从那里驱动相同的服务器。

您可以保存多个配置文件，每个环境（生产、测试、开发）一个，并通过任何命令上的 `--config` 标志在它们之间切换。参见 [设置多个配置](/zh/docs/setup#multiple-configs)。
