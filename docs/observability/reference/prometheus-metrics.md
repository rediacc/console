# Prometheus Metrics

## Tempo Span Metrics

Auto-generated by Tempo from ingested traces. These are the primary metrics used across all dashboards.

### Metric Names

| Metric | Type | Description |
|--------|------|-------------|
| `traces_spanmetrics_calls_total` | counter | Total request/operation count |
| `traces_spanmetrics_latency_bucket` | histogram | Latency distribution (histogram buckets) |
| `traces_spanmetrics_latency_sum` | counter | Cumulative latency (for computing averages) |
| `traces_spanmetrics_latency_count` | counter | Total observation count for latency histogram |
| `traces_spanmetrics_size_total` | counter | Total span size |

### Labels

All span metrics carry these labels:

| Label | Description | Example |
|-------|-------------|---------|
| `service` | Service name | `account-server`, `rdc-cli`, `renet` |
| `span_name` | Operation or route | `POST /api/v1/license/activate` |
| `status_code` | Span status | `STATUS_CODE_OK`, `STATUS_CODE_ERROR` |
| `service_version` | Deployed version | `1.5.0` |
| `deployment_environment` | Environment | `production` |

## Common PromQL Queries

### Error rate by service

```promql
sum(rate(traces_spanmetrics_calls_total{status_code="STATUS_CODE_ERROR"}[5m])) by (service)
```

### Request rate by service

```promql
sum(rate(traces_spanmetrics_calls_total[5m])) by (service)
```

### P95 latency by service

```promql
histogram_quantile(0.95, sum(rate(traces_spanmetrics_latency_bucket[5m])) by (le, service))
```

### Error percentage by service

```promql
(
  sum(rate(traces_spanmetrics_calls_total{status_code="STATUS_CODE_ERROR"}[5m])) by (service)
  /
  sum(rate(traces_spanmetrics_calls_total[5m])) by (service)
) * 100
```

### Top 10 failing routes (last 1h)

```promql
topk(10, sum by (span_name) (increase(traces_spanmetrics_calls_total{status_code="STATUS_CODE_ERROR"}[1h])))
```

## OTLP Pipeline Metrics

Emitted by the OpenTelemetry Collector (Alloy). Use these to monitor the telemetry pipeline itself.

| Metric | Description |
|--------|-------------|
| `otelcol_receiver_accepted_spans_total` | Spans successfully received by the collector |
| `otelcol_receiver_refused_spans_total` | Spans rejected by the collector (backpressure, invalid data) |
| `otelcol_exporter_sent_spans_total` | Spans successfully exported to backends (Tempo) |

A growing gap between accepted and exported spans indicates pipeline backpressure or exporter failures.

## System Metrics (Node Exporter)

Standard node exporter metrics for host resource monitoring.

| Metric | Description |
|--------|-------------|
| `node_cpu_seconds_total` | CPU time in each mode (user, system, idle, etc.) |
| `node_memory_MemTotal_bytes` | Total system memory |
| `node_memory_MemAvailable_bytes` | Available memory |
| `node_filesystem_avail_bytes` | Available filesystem space |
| `node_filesystem_size_bytes` | Total filesystem size |
| `node_network_receive_bytes_total` | Network bytes received |
| `node_network_transmit_bytes_total` | Network bytes transmitted |
| `node_disk_read_bytes_total` | Disk bytes read |
| `node_disk_written_bytes_total` | Disk bytes written |
